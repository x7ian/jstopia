<DocBlock anchor="why-javascript" kind="rule" title="Why JavaScript happened">
A browser needed a way to do things like:
- respond to clicks,
- validate a form,
- update the page without reloading,
- compute values and display results,
- coordinate timing and events.

In other words: the browser needed **a runtime** for logic.

JavaScript became the language that could run inside that environment.
Over time, JavaScript also expanded beyond the browser ‚Äî but it never stopped being shaped by that origin.
</DocBlock>

<DocBlock anchor="three-forces-preview" kind="explanation" title="The three forces: Runtime, Loop, Flux">
These aren‚Äôt ‚Äúmagic words‚Äù ‚Äî they‚Äôre the foundations of how JavaScript actually behaves.

- **Runtime**: the environment where JavaScript runs (browser, Node, etc.)  
- **Loop**: how execution is scheduled (call stack + tasks + microtasks)  
- **Flux**: our metaphor for real flows (control flow, data flow, event flow)

You‚Äôll meet each one properly in the next Prologue topics.
